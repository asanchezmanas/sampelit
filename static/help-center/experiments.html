Experiments Guide

A comprehensive guide to creating, managing, and optimizing A/B tests with Samplit's advanced experimentation engine.

schedule Last updated Dec 26, 2025
timer 15 min read
Optimization v2.4

Understanding Experiments

An experiment in Samplit compares different versions of content to determine which performs best. The platform uses Adaptive Optimization algorithms to automatically allocate more traffic to better-performing variations while the test runs.

Draft

Configuration phase, not yet visible to visitors.

Active

Experiment is live and collecting real-time behavioral data.

Completed

Winner has been declared and the variation is implemented.

Experiment Types

A/B Test

Classic comparison between a Control and a single Variant. Best for quick UI tweaks.

Multi-Element

Test several elements (headings, images, buttons) simultaneously to find the best combination.

Split URL

Send visitors to completely different URLs. Ideal for testing major page redesigns.

A/B/n

Testing up to 10 variations at once to explore a wide range of creative ideas.

Creating an Experiment

  1. Basic Setup: Provide a descriptive name and the target URL.
  2. Method Choice: Choose the Visual Editor for point-and-click changes, or Code Editor for custom scripts.
  3. Define Variations: Each element needs a Control (original) and at least one Variant.
  4. Traffic Rules: Decide what percentage of your total traffic should participate in the test.

Traffic Allocation

bolt

Adaptive Optimization

Unlike traditional tools that split traffic 50/50 throughout the test, Samplit's Multi-Armed Bandit algorithm shifts traffic to winning variations in real-time. This reduces conversion loss during the experimentation phase.

Setting Goals

Goals are the primary metrics you use to define success. You can set a Primary Goal for optimization and multiple Secondary Goals for deeper insights.

Goal Type Trigger Use Case
Click User clicks a specific element CTA buttons, navigation links
Page View User reaches a specific URL Thank you pages, checkout success
Form Submit Successful form submission Lead generation, newsletters
Revenue Transaction value detected E-commerce store optimization

Best Practices

  • Wait for Significance: Never declare a winner before reaching at least 95% confidence and 100 conversions per variation.
  • One Hypothesis: Focus each experiment on a single clear question (e.g., "Does scarcity-focused text increase urgency?").
  • Device Sensitivity: Always review your variations on mobile. A winning desktop variation might perform poorly on smaller screens.
  • Document Everything: Keep a log of your experiments to build an organizational knowledge base of what works for your audience.
