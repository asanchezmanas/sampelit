# Case Studies: Our Experiments

Transparent analysis of experiments we run on our own properties.

## Purpose
- Build trust through transparency
- Show real data (including failures)
- Demonstrate the value of testing
- Provide actionable insights

## Format for Each Case Study

```markdown
# [Experiment Name]

## Summary
- **What we tested:** [Element]
- **Duration:** [X days]
- **Traffic:** [X visitors]
- **Result:** [Winner / Inconclusive / Loser]
- **Impact:** [+X% / No change]

## Background
Why we ran this test. What problem we were trying to solve.

## Hypothesis
"We believe that [change] will [outcome] because [reasoning]."

## Variants
- Control: [Description]
- Variant A: [Description]
- Variant B: [Description] (if applicable)
```

## Guidelines

1. **Never cherry-pick results** - Publish everything, including inconclusive tests
2. **Be honest about limitations** - Sample size, external factors, etc.
3. **Make it actionable** - What can readers apply to their own sites?
4. **Link to methodology** - Reference the testing approach for credibility
5. **Update if needed** - If long-term results differ, add an update
